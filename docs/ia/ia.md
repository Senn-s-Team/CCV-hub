# ccv-hub 信息架构

## 1. 文档目的

本文定义 `ccv-hub` 自用 MVP 的信息架构，用于把已确认的产品范围压缩成可直接指导实现的页面骨架、主流程和状态边界。本文只描述页面如何组织信息，不展开系统实现、接口协议和发现机制细节。

## 2. 范围边界

### 2.1 范围内

- 本机正在运行的 `cc-viewer` 实例
- 单页总览
- 顶部栏“启动新实例”动作
- 路径输入弹窗
- 项目名筛选
- 一键打开实例
- 复制实例链接
- 默认按最近启动时间降序排序

### 2.2 范围外

- 已退出或历史实例
- 远程机器实例
- 实例详情页
- 高级筛选与分组
- 团队协作与权限能力
- 对 `cc-viewer` 源码的修改

## 3. 核心对象

### 3.1 Instance

页面中的核心对象是运行中的 `cc-viewer` 实例。

每个实例在页面中至少暴露以下信息：
- 项目名
- 项目路径
- 实例链接
- 运行状态
- 启动时间
- 发现来源

## 4. 页面清单

### 4.1 总览页

总览页是 MVP 的唯一主页面，承担查看、筛选、打开和复制链接四类动作。

### 4.2 启动弹窗

点击顶部栏“启动新实例”后，页面弹出路径输入框。该弹窗用于输入项目路径并触发统一入口启动。

## 5. 页面结构

```text
CCV Hub
├── 顶部栏
│   ├── 页面标题
│   ├── 启动新实例
│   └── 刷新按钮
├── 筛选区
│   └── 项目名关键字输入框
└── 实例列表区
    ├── 项目名
    ├── 项目路径
    ├── 实例链接
    ├── 运行状态
    ├── 启动时间
    ├── 发现来源
    ├── 打开按钮
    └── 复制链接按钮
```

## 6. 信息层级

### 6.1 列表主信息

- 项目名

项目名是用户识别实例的第一层信息，应当在视觉上优先。

### 6.2 列表次信息

- 项目路径
- 实例链接

这两项帮助用户确认实例属于哪个目录、实际打开的目标地址是什么。

### 6.3 列表元信息

- 运行状态
- 启动时间
- 发现来源

这些字段用于帮助用户判断实例是否可信、是否是最新启动，以及该实例如何被登记到总览页。

### 6.4 列表动作

- 打开
- 复制链接

动作直接附着在列表项上，避免进入二级页面。

## 7. 关键用户流

### 7.1 打开已有实例

进入总览页 → 查看实例列表 → 定位目标项目 → 点击打开 → 在新标签页进入对应实例

### 7.2 复制实例链接

进入总览页 → 定位目标项目 → 点击复制链接 → 获取实例地址

### 7.3 按项目名筛选

进入总览页 → 在筛选框输入关键字 → 列表实时或低延迟刷新 → 保留命中的实例项

### 7.4 启动新实例

进入总览页 → 点击“启动新实例” → 弹出路径输入框 → 输入项目路径 → 触发统一入口启动 → 新实例出现在列表中

## 8. 状态模型

### 8.1 loading

页面首次加载时展示读取中的过渡状态。

### 8.2 list-ready

页面成功拿到实例数据并展示列表。

### 8.3 empty

当前没有运行中的实例时，页面展示空状态，并明确指向“启动新实例”。

### 8.4 launch-failed

输入路径后启动失败时，弹窗或页面应明确提示失败结果。

### 8.5 discovery-error

当实例登记或列表读取失败时，页面展示错误状态，并保留刷新动作。

## 9. 排序与筛选规则

- 默认按最近启动时间降序排序
- 第一版只支持按项目名关键字筛选
- 第一版不提供高级筛选、组合筛选或自定义排序

## 10. MVP 约束

- 只保留一个主页面和一个启动弹窗
- 列表直接承载主要操作
- 不引入实例详情页
- 不展示历史实例
- 不把 system/api 设计混入 IA 文档
