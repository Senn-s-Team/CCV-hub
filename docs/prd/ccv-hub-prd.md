# ccv-hub 产品需求文档

## 1. 产品概述

ccv-hub 是一个独立网页，用于汇总同一台机器上正在运行的 cc-viewer 实例，并提供统一列表、一键打开、复制链接等入口能力。它面向同时维护多个项目、需要频繁切换不同 cc-viewer 实例的用户，目标是把分散的实例入口收敛成一个稳定、直接、低摩擦的本地控制台。

## 2. 背景与问题

当前 cc-viewer 已支持单实例运行，也具备工作区注册与 Electron 多项目能力。但在“多个独立进程分别运行”的场景下，用户仍需自己记忆端口、窗口和项目对应关系。实例入口分散，切换成本高，实例状态缺少统一视图。

## 3. 产品目标

### 3.1 目标

1. 在一个页面内展示本机当前所有正在运行的 cc-viewer 实例。
2. 让用户可以从统一入口快速打开目标实例。
3. 通过统一入口启动实现实例自动登记。
4. 让页面只呈现运行中实例，保持信息密度和认知负担可控。

### 3.2 非目标

1. 不展示已退出或历史实例。
2. 不修改 cc-viewer 源码。
3. 不做跨机器实例聚合。
4. 不做实例生命周期管理面板之外的复杂控制能力。
5. 不做权限系统、团队协作、云端同步。

## 4. 目标用户

主要用户是在同一台机器上并行运行多个项目的开发者，需要在多个 cc-viewer 实例之间快速切换，并希望避免手工记录端口与链接。

## 5. 使用场景

### 5.1 典型场景

1. 用户同时打开多个项目，每个项目各自运行一个 cc-viewer 实例。
2. 用户通过统一入口启动某个项目的 cc-viewer。
3. ccv-hub 自动登记新实例。
4. 用户打开总览页，看到所有运行中实例。
5. 用户点击列表项，进入对应实例页面。

### 5.2 用户收益

1. 减少记忆端口和链接的成本。
2. 降低多项目切换摩擦。
3. 用统一视图确认哪些实例处于运行状态。

## 6. 产品范围

### 6.1 第一版范围

1. 独立网页总览页。
2. 运行中实例列表。
3. 使用 `ccv-hub` 作为统一入口启动命令。
4. 实例自动登记。
5. 一键打开实例。
6. 复制实例链接。
7. 按项目名筛选。
8. 默认按最近启动时间降序排序。

### 6.2 后续范围

1. 收藏常用实例。
2. 最近打开记录。
3. 分组视图。
4. 轻量健康状态细节。
5. 更多运行元信息展示。

## 7. 关键概念

### 7.1 实例

实例指一条正在运行中的 cc-viewer 进程及其可访问 Web 入口。

### 7.2 统一入口启动

统一入口启动指用户通过 `ccv-hub` 命令启动 cc-viewer，而不是直接从零散命令自行拉起进程。该入口负责自动登记实例元数据。

### 7.3 自动登记

自动登记指实例启动完成后，其基础信息被记录到 ccv-hub 的本地实例目录中，供总览页读取。

## 8. 功能需求

### 8.1 实例总览页

总览页必须展示当前本机所有运行中实例。

每个实例至少展示以下字段：
- 项目名
- 项目路径
- 实例链接
- 运行状态
- 启动时间
- 发现来源

### 8.2 一键打开

用户点击实例项或打开按钮后，系统应在浏览器新标签页中打开目标实例链接。

### 8.3 复制链接

用户可一键复制实例链接，用于手动分享或粘贴到其他工具。

### 8.4 筛选与排序

第一版支持：
- 按项目名关键字筛选
- 默认按最近启动时间降序排序
- 按运行状态排序

### 8.5 自动登记

通过统一入口启动的新实例，必须自动出现在总览页中。

### 8.6 只展示运行中实例

默认视图只显示运行中实例。实例退出后，应在合理时间内从列表移除，或收敛为短暂可见的退出状态后移除。

## 9. 数据模型

第一版实例记录至少包含：

- `id`
- `projectName`
- `projectPath`
- `url`
- `port`
- `pid`
- `status`
- `source`
- `startedAt`
- `lastSeen`

## 10. 页面结构

```text
CCV Hub
├── 顶部栏
│   ├── 页面标题
│   ├── 启动新实例
│   └── 刷新按钮
└── 实例列表
    ├── 筛选框
    ├── 项目名
    ├── 项目路径
    ├── 链接
    ├── 状态
    ├── 启动时间
    ├── 打开按钮
    └── 复制链接按钮
```

## 11. 交互要求

1. 页面首次打开时，自动加载当前运行中实例。
2. 顶部栏提供“启动新实例”动作，点击后直接弹出路径输入框。
3. 新实例登记后，列表应自动刷新或支持低延迟刷新。
4. 打开实例操作必须直接、明确，避免多层跳转。
5. 当没有运行中实例时，页面展示空状态，并指向统一入口启动。
6. 当实例登记异常或链接失效时，页面展示明确错误状态。

## 12. 状态定义

### 12.1 running

实例已启动且链接可访问。

### 12.2 stale

实例最近曾登记，但当前心跳或状态校验已超时。第一版可将其视为过渡状态，并尽快从运行列表移除。

### 12.3 exited

实例已退出。第一版默认不展示 exited 状态项。

## 13. 非功能需求

1. 页面应在本地环境快速打开。
2. 实例列表读取应保持轻量，避免高频重计算。
3. 发现与登记机制应优先追求稳定和可解释。
4. 页面交互应在桌面浏览器下可用。

## 14. 依赖与约束

1. ccv-hub 独立开发于 `ccv-hub/`，不修改 `cc-viewer/` 源码。
2. 第一版接受“统一入口启动 + 自动登记”作为主发现机制。
3. 当前范围限定为本机实例，不包含远程实例。

## 15. 验收标准

### 15.1 功能验收

1. 用户通过统一入口启动 1 个 cc-viewer 实例后，总览页可见该实例。
2. 用户通过统一入口连续启动多个项目后，总览页可同时展示多个实例。
3. 点击任一实例的打开按钮，可进入对应实例页面。
4. 点击复制链接按钮，可得到正确实例链接。
5. 用户输入项目名关键字后，列表按预期筛选。
6. 实例退出后，列表会在可接受时间内更新。

### 15.2 体验验收

1. 列表信息足以区分多个同类项目。
2. 常见操作路径不超过两步。
3. 空状态和错误状态可被直接理解。

## 16. 风险与待确认项

### 16.1 已知风险

1. 实例退出后的移除时机若处理不稳，页面会出现短暂脏数据。
2. 若统一入口之外仍存在大量手工启动行为，第一版覆盖率会下降。

### 16.2 待确认项

当前无阻塞性待确认项。

## 17. 版本定义

### V1

- 独立网页
- 统一入口启动
- 自动登记
- 运行中实例总览
- 一键打开
- 复制链接
- 基础筛选与排序
